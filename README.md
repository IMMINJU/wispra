# wispra

영어와 한국어를 **양방향으로** 실시간 통역해 자막으로 띄우는 웹앱. 마이크 하나로
두 사람의 대화를 듣고, **말한 언어를 자동 감지**해 상대 언어 자막을 OpenAI
`gpt-realtime-translate`로 띄웁니다. **스포티파이 가사처럼** 현재 문장이 화면
중앙에 크게 뜨고 지난 줄은 위로 흐려지며 흘러가며, **누가 말했는지 색으로
구분**됩니다(영어→초록, 한국어→파랑). 정지하면 **원본 녹음(.webm)** 과
**통역 대화 전문(.txt, 방향 라벨 포함)** 이 자동 다운로드됩니다.

## 동작 구조

```
브라우저(마이크 1개 캡처 → 두 세션에 동시 전송 + 자막 + 녹음)
        │  ① GET /api/token  →  ko·en 두 임시 토큰 발급 (실제 키는 서버에만)
        │  ② 토큰마다 OpenAI에 직접 WebSocket 연결 (subprotocol 인증)
        ▼
   OpenAI Realtime  × 2
     ├─ 세션 A: 출력 ko  ⇐ 영어가 들리면 번역(=영어 화자)
     └─ 세션 B: 출력 en  ⇐ 한국어가 들리면 번역(=한국어 화자)
```

- **방향 자동 감지의 원리**: 같은 마이크 오디오를 두 세션에 똑같이 보냅니다.
  OpenAI는 입력 언어를 자동 감지하고 **"이미 출력 언어인 발화는 번역하지 않으려
  하므로"**, 영어를 말하면 ko-세션만, 한국어를 말하면 en-세션만 자막을 내보냅니다.
  그 침묵/활성 차이로 방향을 가립니다.
- **API 키는 서버(Vercel 함수 / 로컬 Node)에만** 있고, 브라우저엔 1분짜리 임시
  토큰만 전달됩니다.
- 마이크(`getUserMedia`)는 **https 또는 localhost에서만** 동작합니다. 폰에서
  쓰려면 Vercel 배포(https)가 필요합니다.

## 빠른 시작 (로컬, 사람·마이크 없이도 확인 가능)

```bash
npm install
cp .env.example .env        # .env 안에 OPENAI_API_KEY 채우기
npm start
```

http://localhost:3000 을 브라우저에서 열고:

- **▷ 데모 보기 (샘플 음성)** — 번들된 **영↔한 짧은 대화**(~14초)를 스피커로
  **들려주면서** 자막을 시연합니다. 자막은 방향 토글(기본 **EN→KO**)을 따르므로
  영어 차례는 한국어 자막이 뜨고, 토글을 KO→EN으로 바꾸면 영어 자막을 볼 수
  있어요. 마이크·발화자 없이 동작/디자인을 바로 볼 수 있습니다. (샘플은
  `npm run sample:convo` 로 생성하며 `public/sample.pcm` 로 커밋되어 배포에도 포함)
- **▶ 시작하기** — 실제 마이크로 두 사람의 영어·한국어 대화를 양방향 통역합니다.

> 폰에서 `http://노트북IP:3000` 로 접속하면 https가 아니라 마이크가 막힙니다.
> 폰 테스트는 아래 Vercel 배포로 하거나, `ngrok http 3000` 같은 터널로 https
> 주소를 받으세요.

## 배포 (Vercel — 폰에서 쓰려면 이 방법)

```bash
npm i -g vercel
vercel                      # 배포 (질문에 기본값 엔터)
vercel env add OPENAI_API_KEY   # 키 입력 (Production 선택)
vercel --prod               # 키 반영해 재배포
```

나오는 `https://...vercel.app` 주소를 **폰 브라우저에서 열고** → 시작 → 마이크
허용. 폰을 가로로 눕히고 글자를 키우면 가독성이 가장 좋습니다.

> **데모 보기는 배포본에서도 동작합니다** — `public/sample.pcm`(번들 샘플 음성)이
> 레포에 커밋되어 정적 파일로 함께 배포되기 때문입니다. 샘플을 바꾸려면
> `npm run sample:convo` 로 다시 생성한 뒤 커밋하세요.

## 사용 / 다운로드

- **시작/데모** → 통역 자막이 흐름. 영어로 말하면 **초록**, 한국어로 말하면
  **파랑** 줄로 표시됨. 상단 바는 3초 뒤 자동으로 숨음(화면 탭하면 다시 표시).
  - **데모 보기**는 번들된 **영↔한 짧은 대화**(~14초)를 스피커로 들려주며 자막을
    시연합니다. 자막은 토글 방향(기본 EN→KO)을 따라요. (`npm run sample:convo` 로 생성)
- **방향 버튼 (EN→KO ⇄ KO→EN)** — **화면 하단 중앙에 항상 떠 있고**(상단 바가
  숨어도 안 사라짐) 탭하면 두 방향이 전환됩니다. 오디오는 늘 두 세션에 가지만
  **표시만** 한쪽으로 거릅니다 — 한 방향만 보이니 같은 언어 에코와 단어 단위
  뒤섞임이 없어요. 말하는 사람이 바뀌면 누르세요.
- **A− / A+** 글자 크기, **⛶** 전체화면.
- **정지** → 자동 다운로드:
  - `녹음-YYYYMMDD-HHMM.webm` — 원본 마이크 음성
  - `대화-YYYYMMDD-HHMM.txt` — 통역 대화 전문 (각 줄에 `[EN→KO]`/`[KO→EN]` 방향
    라벨). 방향 버튼과 무관하게 **양방향 전체**가 저장됩니다.

## 검증 하니스 (선택)

마이크·브라우저 없이 번역 파이프라인을 터미널에서 검증합니다.

```bash
# 단방향 (EN→KO) 파이프라인
npm run sample        # 영어 음성 샘플 생성 (OpenAI TTS → harness/sample.pcm)
npm run verify        # 그 샘플을 번역 WS에 스트리밍 → 한국어 번역 + 지연(ms)

# 양방향 검증 — 같은 오디오를 ko/en 두 세션에 동시에 흘려, 어느 쪽이 번역하고
# 어느 쪽이 침묵하는지로 "방향 자동 감지"를 확인 (라이브 실측 완료)
npm run sample:ko       # 한국어 음성 샘플 생성 (→ harness/sample-ko.pcm)
npm run verify:bidi:ko  # 한국어 입력 → EN 세션이 번역, KO 세션 침묵이면 ✅
npm run verify:bidi:en  # 영어 입력  → KO 세션이 번역, EN 세션 침묵이면 ✅

# 브라우저 데모용 영↔한 번갈아 대화 샘플 (→ public/sample.pcm 로도 복사)
npm run sample:convo
```

`SAMPLE_TEXT="..." npm run sample` (또는 `sample:ko`) 로 다른 문장도 테스트 가능.

> 양방향 동작은 **라이브 실측 완료** — 한→영/영→한 번역, 방향 토글 필터,
> 브라우저 데모 모두 실제 오디오로 확인. OpenAI 공식 문서로도 뒷받침됨(입력
> 언어 자동 감지, 70+ 입력/13 출력 언어, 방향마다 세션 분리가 권장 패턴).

## 파일

- `public/index.html` — 자막 UI 전부 (마이크 캡처 → 두 세션 fan-out·세그먼트
  렌더링·화자 색·녹음·다운로드)
- `public/pcm-worklet.js` — 마이크 오디오를 24kHz PCM16으로 변환
- `api/token.js` — Vercel 서버리스 함수: ko·en **두** 임시 토큰 발급
- `server.js` — 로컬 개발 서버 (두 토큰 발급 + 정적 서빙, Vercel 동작을 미러)
- `harness/` — 헤드리스 검증/샘플 스크립트 (`verify.js` 단방향, `verify-bidi.js`
  양방향, `make-convo-sample.js` 데모용 영↔한 번갈아 대화 샘플)
- `vercel.json` — 배포 설정

## 비용

양방향은 세션을 **2개** 돌리므로 단방향의 약 2배입니다. 분당 약 **$0.07**
(세션당 ~$0.034 × 2, 오디오 길이 기준). 30분 대화 ≈ $2, 1시간 ≈ $4.
검증 하니스는 회당 몇 센트(TTS + 번역).

## 현장 팁

- **1:1 대면 대화**용입니다. 마이크 하나를 두 사람 **사이**에 두세요(테이블 위 등).
- 두 사람이 **동시에** 말하면 두 색 줄이 단어 단위로 뒤섞일 수 있어요. **방향
  버튼(EN→KO ⇄ KO→EN)** 으로 지금 말하는 사람 쪽을 켜두면 깔끔합니다.
- 행사·미팅 전에 한 번 리허설로 지연·품질을 확인하세요.
- iOS Safari / 안드로이드 Chrome 모두 동작. 폰 자동잠금을 잠시 꺼두면 화면이 안 꺼집니다.

## 커스터마이징

- **언어 쌍**: 기본 `ko`↔`en`. 환경변수 `LANG_A` / `LANG_B` 로 바꿉니다
  (예: 일본어↔영어면 `LANG_A=ja`, `LANG_B=en`). 클라이언트는 `index.html` 의
  `SESS` 배열이 두 세션의 출력 언어를 정합니다. ⚠️ 화자 색은 텍스트의
  **스크립트(한글/로마자)** 로 판정하므로(`speakerOf`), 스크립트를 공유하는
  언어 쌍(예: es↔en)에서는 색 구분이 안 됩니다 — 그 경우 별도 언어감지 필요.
- **방향 토글**: `DIR_MODES` / `DIR_SHOWS` (EN→KO ⇄ KO→EN). 하단 중앙 고정 버튼.
- **화자 색**: `index.html` CSS의 `--spk-en`(초록) / `--spk-ko`(파랑).
- **`MAX_LINES`** — 화면에 남는 자막 줄 수.
- **`BASE_CPS` / `MAX_CPS`** — 자막이 풀리는 속도(초당 글자 수). 큐가 밀리면
  `MAX_CPS`까지 빨라져 음성을 따라잡습니다.

## 알려진 한계

- OpenAI의 한국어 번역 출력이 가끔 두 단어를 공백 없이 붙여 보냅니다
  (예: `모델은작은`). 원본 출력 자체의 문제로, 빈도는 낮습니다.
- **같은 언어 에코**: `gpt-realtime-translate`가 가끔(매번은 아님) 출력 언어와
  같은 발화를 침묵 대신 그대로 받아적습니다 — 실측으로 두 세션 모두에서 확인됨.
  두 방향이 동시에 활성이면 두 색 줄이 단어 단위로 뒤섞일 수 있는데, 기본 동작이
  **한 방향만 표시**라 평소엔 안 보입니다(방향 버튼으로 전환). (라이브 검증 완료)
- 폰 실기기 + 현장 마이크 환경은 별도 리허설로 확인하세요.
- **라이브 실측은 아직 미완료**: OpenAI 크레딧 소진으로 한→영/양방향 동시
  동작을 실제 오디오로 검증하지 못했습니다. 설계는 OpenAI 공식 문서로 확인됨.
  크레딧 충전 후 `harness/verify-bidi.js`(있다면) 또는 데모로 확인하세요.
